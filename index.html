Project Resilience

Self-Healing 3-Tier Architecture on AWS

The Problem

Manual remediation of Nginx failures was causing 15-minute downtimes during off-hours. The traditional approach required human intervention to detect, diagnose, and restart failed services, leading to poor user experience and potential SLA breaches.

The Solution

Implemented an event-driven self-healing architecture that automatically detects and recovers from failures:

  • ALB health checks continuously monitor EC2 instances running Nginx
  • CloudWatch Alarms trigger on unhealthy target status
  • EventBridge rules invoke Lambda functions for automated remediation
  • Auto Scaling Group replaces unhealthy instances automatically
  • Discord webhooks provide real-time incident notifications

Architecture Diagram

Infrastructure as Code

Terraform configuration for the Auto Scaling Group:

Terraform asg.tf 
resource "aws_autoscaling_group" "web_asg" {
  name                = "resilience-web-asg"
  desired_capacity    = 2
  max_size            = 4
  min_size            = 1
  target_group_arns   = [aws_lb_target_group.web_tg.arn]
  vpc_zone_identifier = aws_subnet.private[*].id

  launch_template {
    id      = aws_launch_template.web_lt.id
    version = "$Latest"
  }

  health_check_type         = "ELB"
  health_check_grace_period = 300

  tag {
    key                 = "Name"
    value               = "resilience-web-server"
    propagate_at_launch = true
  }

  lifecycle {
    create_before_destroy = true
  }
}

The Results

90% Reduction in MTTR Mean Time To Recovery
<30s Detection Time From 15 minutes manual
99.9% Uptime Achieved Singapore region (ap-southeast-1)
0 Manual Interventions Fully automated recovery

Cloud Janitor

Automated Resource Cleanup & Cost Optimization

The Problem

Orphaned resources and forgotten development environments were accumulating, causing unnecessary cloud spend and security risks. Manual audits were time-consuming and often missed resources across multiple regions.

The Solution

Built a serverless cleanup automation system that identifies and removes unused resources:

  • EventBridge scheduled rules trigger daily cleanup scans
  • Lambda functions identify resources based on tags and usage patterns
  • SNS notifications alert before resource deletion
  • DynamoDB tracks cleanup history and exemptions
  • CloudFormation stacks auto-delete after TTL expiration

Architecture Diagram

Infrastructure as Code

Terraform configuration for the cleanup Lambda:

Terraform lambda.tf 
resource "aws_lambda_function" "cloud_janitor" {
  function_name = "cloud-janitor-cleanup"
  role          = aws_iam_role.janitor_role.arn
  handler       = "index.handler"
  runtime       = "python3.11"
  timeout       = 300
  memory_size   = 256

  filename         = data.archive_file.lambda_zip.output_path
  source_code_hash = data.archive_file.lambda_zip.output_base64sha256

  environment {
    variables = {
      DRY_RUN         = "false"
      SLACK_WEBHOOK   = var.slack_webhook_url
      RETENTION_DAYS  = "7"
    }
  }

  tags = {
    Project     = "CloudJanitor"
    Environment = "Production"
  }
}

The Results

40% Cost Reduction Monthly cloud spend
500+ Resources Cleaned In first month
24/7 Automated Scanning Zero manual effort
5 min Audit Time Down from 4 hours
